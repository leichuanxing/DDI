# 合力数据 DDI 管理系统 v1.0 - 完整功能介绍文档

## 一、项目概述

**项目名称**: 合力数据 DDI 管理系统

**技术栈**: Python 3.11+ / Django 4.x / Bootstrap 5 / Chart.js / SQLite 3

**项目路径**: `/opt/codebuddy/ddi_system/`

本系统是一个轻量级网络资源管理平台，统一管理企业内部的 **IP 地址资源（IPAM）**、**DHCP 地址分配** 和 **DNS 解析记录**。内置轻量级 DNS/DHCP 服务器实现和网络探测引擎，提供从 IP 资源规划到服务可达性验证的全流程管理能力。

#### 探测引擎 (`scanner.py`)

**核心类 (511 行):**

| 类 | 功能 | 特性 |
|----|------|------|
| `PingScanner` | Ping 探测器 | 跨平台(Windows/Linux/Mac)、批量并发(默认50线程)、解析延迟/丢包/TTL |
| `PortScanner` | TCP 端口扫描器 | 连接扫描模式、Banner 抓取、常见服务端口自动识别(40+服务)、端口范围解析(`1-1000`)、并发100线程 |
| `ARPScanner` | ARP 扫描器 | 读取系统 ARP 表、MAC 厂商 OUI 识别(50+厂商) |
| `DNSScanner` | DNS 反向解析器 | 反向 DNS 查询、批量并发(30线程) |
| `NetworkScanner` | 综合扫描器 | 整合以上所有能力，支持两阶段扫描(Ping→端口)、进度回调 |

**支持的扫描类型:**
- **ping**: 仅 ICMP Ping 检测存活主机
- **port**: Ping + TCP 端口扫描
- **arp**: Ping + ARP 表查询(MAC/厂商)
- **full**: 综合扫描(Ping + DNS反解 + 端口 + ARP)

## 五、核心业务规则

1. **子网必须符合标准 CIDR 格式** (`192.168.1.0/24`)
2. **同一 IP 在同一时刻只能有一个有效占用状态**
3. **DHCP 地址池起止地址不能超出子网范围**，起始 ≤ 结束
4. **DNS A 记录绑定的 IP 必须合法且存在**
5. **已保留地址不可重复分配**
6. **网关地址默认标记为保留状态** (创建子网自动生成 IP 时)
7. **所有删除操作需二次确认** (confirm_delete.html 模板)
8. **DNS 记录启用受探测任务影响** — 关联了 probe_port 的记录，只有对应探测任务状态为 reachable 时才会被 DNS 服务器返回给客户端
9. **DHCP/DNS 服务需要 root 权限** 绑定 53/67 端口
10. **用户密码重置统一为 `Admin@123`**

## 七、系统特色功能总结

### 7.1 智能 DNS 选路与探测联动
- DNS 记录可关联**探测端口**，创建时自动探测目标 IP:Port 可达性
- **两级匹配策略**: 精确(target+port) + 端口回退(port only)
- DNS 解析时根据探测任务**实时状态**动态决定返回哪条记录
- 探测不可达的记录自动跳过，实现**故障自动切换**

### 7.2 内置 DNS/DHCP 服务器
- **纯 Python 实现**，无需依赖 BIND/dhcpd
- DNS 服务器支持: 本地区域查询、上游转发、查询缓存、NXDOMAIN 处理、完整日志
- DHCP 服务器支持: DORA 标准流程、Option 配置、地址池管理、租约持久化
- 通过 **Web UI 一键启停**，配置热加载

### 7.3 网络探测引擎
- **Ping 探测**: 跨平台、批量并发(默认50线程)、延迟/丢包/TTL 统计
- **端口扫描**: TCP 连接扫描、Banner 抓取、40+ 服务自动识别、最大 1000 端口/次
- **ARP 扫描**: 系统 ARP 表读取、50+ 厂商 OUI 识别
- **DNS 反解**: 批量反向 DNS 查询
- **异步执行**: 后台线程运行，AJAX 轮询进度
- **一键导出**: CSV 格式扫描报告

### 7.4 全链路审计
- 所有增删改操作自动记录 **OperationLog**
- 登录成功/失败记录 **LoginLog**
- DNS 解析查询记录 **DNSQueryLog** (最多10000条，自动滚动清理)
- 支持按模块/操作类型/关键字筛选查询
